原型

原型：每一个对象都有一个prototype(原型)属性，这个属性是一个指针，指向一个对象，这个对象包含可以由特定类型的所有实例共享的属性和方法

每个原型对象都有一个constructor(构造函数)属性，这个属性是一个指向该原型对象构造函数的指针

当调用构造函数创建一个新实例后，该实例的内部将包含一个指向构造函数的原型对象的指针(_proto_)

可以通过isPrototypeOf来确定对象之间是否存在这种关系，或通过Object.getPrototypeOf()来获取prototype的值

每当代码读取某个对象的某个属性时，都会执行一次搜索，首先从对象实例本身开始。如果在实例中找到了具有给定名字的属性，则返回该属性的值；如果没有找到，则继续搜索指针指向的原型对象，如果找到了，则返回该属性的值

当为实例添加一个属性时，这个属性就会屏蔽原型对象中保存的同名属性，并不会修改原型中的属性

当重写整个原型对象后，就会切断构造函数与最初原型之间的联系，constructor的指向也会改变

创建自定义类型的最常见方式，就是组合使用构造函数模式和原型模式。将实例属性定义在构造函数内，原型模式用于定义方法和共享的属性，此时的每一个实例都会有自己的一份实力属性的副本，同时又共享着对方法的引用

原型链和继承

每个构造函数都有一个原型对象(constructor.prototype)，原型对象都包含一个指向构造函数的指针(constructor)，实例都包含一个指向原型对象的内部指针(__proto__)

实例的__proto__指针指向其构造函数的prototype属性

当让原型对象等于另一个类型的实例，此时的原型对象将包含指向另一个原型的指针，另一个原型中也包含指向另一个构造函数的指针。当多个原型相连，就构成了实例与原型的链条，这就是原型链。

subType.prototype = new SuperType()

继承就是通过创建superType的实例，并将该实例赋给subType.prototype实现的。实现的本质是重写原型对象。

var instance = new SubType()

此时的实例指向SubType的原型，而SubType的原型又指向SuperType的原型。

通过原型链实现继承时，不能使用对象字面量创建原型方法，这样会重写原型链

当使用重写原型链的方式继承时，继承SuperType的属性和方法将全部定义在SubType的原型中，此时SubType的所有实例将共享这些属性和方法

继承的最佳实践

这样实例就可以拥有私有属性和共享方法

缺点：会调用两次SuperType()，第一次重写原型时，SuperType中的属性会继承到SubType的原型中，调用SubType()时，又将SuperType的属性继承到构造函数中，屏蔽了原型中的同名属性。此时会有两组同名属性，一组在实例上，一组在原型上。

改进

ES6中的类

ES6类机制的核心是关键词class，表示一个块，其内容定义了一个函数原型的成员

  • constructor指定函数的签名以及函数体内容(相当于ES5中的构造函数)
  • 在class定义体内部不用逗号分隔成员
  • class SuperType的SuperType调用必须通过new来实现
  • 函数声明是提升的，而class不是，在实例化一个class之前必须先声明他
  • 全局作用域中的class SuperType创建了这个作用域的一个词法标识符SuperType，但是和function SuperType不一样，并没有创建一个同名的全局对象属性

extends和super

extends用来在两个函数之间建立prototype连接

在构造器中，super自动指向父构造器(相当于ES5在子类的构造函数中调用SuperType.call(this))

在方法中，super会指向父对象，这样就可以访问其属性和方法了。

子类构造器

对于类和子类来说，构造器并不是必须的，如果省略的活那么两者都会自动提供一个默认构造器，默认子类构造器自动调用父类的构造器并传递所有参数

子类构造器中调用super()之后才能访问this

可以认为创建/初始化实例this的实际上是父构造器
